COMMMAND: crunch [MIN_LEN] [MAX_LEN] [SYMBOLS] -o [FILE_NAME]

FUNCTION: generate a file of passwords by permunations and combinations of symbols.
FIELDS: 
	[MIN_LEN] : minimal lenght of a password.
	[MAX_LEN] : maximal lenght of a password.
	[SYMBOLS] : a string of symbols we want generated password to contain.
	[FILE_NAME] : name of file to write generated passwords.

KEYS: -o : Used to specify the location of the output dictionary file.
      -b : Specifies the maximum number of bytes to write to the file. This size can be specified in KB, MB or GB, but must be used with the -o START option.
      -t : Sets the special format to use.
      -l : This option is used to identify some characters of the placeholder when the -t option specifies @, % or ^.

EXAMPLE: crunch 8 10  ASDFGHJKL123456789 -o passwords.txt
